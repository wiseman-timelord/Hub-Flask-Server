✔️ Example of how JavaScript talks to Flask
JavaScript (browser)

async function uploadFile() {
    const fileInput = document.getElementById('file');
    const formData = new FormData();
    formData.append("file", fileInput.files[0]);

    const response = await fetch("/upload", {
        method: "POST",
        body: formData
    });

    const result = await response.json();
    console.log(result);
}

Flask (server)

from flask import Flask, request, jsonify
import os

app = Flask(__name__)
UPLOAD_FOLDER = "sites"

@app.route('/upload', methods=['POST'])
def upload():
    file = request.files['file']
    username = "bob"  # you will fetch this from session later
    user_folder = os.path.join(UPLOAD_FOLDER, username)
    os.makedirs(user_folder, exist_ok=True)

    file.save(os.path.join(user_folder, file.filename))
    return jsonify({"status": "ok", "file": file.filename})

✔️ What this means

    Your front-end JavaScript handles the UI and sends requests.

    Flask does all the actual work (saving files, routing, creating folders).

    HTML/CSS/JS alone cannot create folders or store files — Flask must do it.

✔️ Flask does not have a UI for configuration

Flask is code-based, not UI-based.
Everything you want Flask to do, you write as Python code (called routes, functions, and configuration variables).

There is:

    ❌ No graphical interface

    ❌ No control panel

    ❌ No “click to configure” UI

Instead, you write Python scripts that define how the backend behaves.
✔️ How the pieces fit together
1. JavaScript (Front-End)

This is the code running in the user's browser.

You write functions like:

    uploadFile()

    createAccount()

    loadUserWebsite()

These functions send HTTP requests to Flask using fetch().

Example (front-end JavaScript):

fetch("/upload", {
    method: "POST",
    body: formData
});

2. Flask (Back-End Python)

This is where you write Python functions that respond to the front-end.

Example (Flask):

@app.route('/upload', methods=['POST'])
def upload():
    # Save file
    return {"status": "ok"}

These are called routes.

Flask has configuration variables, but you set them inside Python, like:

app.config['UPLOAD_FOLDER'] = 'sites/'

No GUI — just code.
